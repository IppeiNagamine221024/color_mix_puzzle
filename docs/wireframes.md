# いろあわせパズル — ワイヤーフレーム

要件定義: [requirements.md](./requirements.md)

初期リリースは **メイン画面** と **ゲーム画面** の2画面。クリア・ゲームオーバーは専用画面を設けず、短いフィードバック後にメイン画面へ戻る想定。

凡例:

```
[ ボタン ]     タップ可能な UI
( 要素 )       表示のみ
┌─┐            枠・エリア
■              色ブロック（例）
░              お邪魔ブロック
·              空きマス
──             広告枠
```

---

## 1. 画面一覧

```
┌─────────────┐         ┌─────────────┐
│ メイン画面   │ ──────→ │ ゲーム画面   │
│（ステージ選択）│ ←────── │             │
└─────────────┘  戻る    └─────────────┘
     │ クリア / GO
     └──────────────────→（メインへ）
```

| 画面 | 役割 |
|------|------|
| メイン画面 | ステージ選択、スタミナ、広告、続きから |
| ゲーム画面 | プレイ本体（バナー広告なし） |

## 2. メイン画面（ステージ選択）

### 2.1 通常時

```
┌──────────────────────────────────────┐
│ いろあわせパズル                        │  ← タイトル / ロゴ
├──────────────────────────────────────┤
│  ♥ ♥ ♡        あと 7:32 で 1 回復      │  ← スタミナ 2/3、回復タイマー
│                                      │
│  [ ▶ 広告でスタミナ回復 ]  (今日 2/5)   │  ← 上限未満時のみ表示
├──────────────────────────────────────┤
│                                      │
│  [ ▶ ステージ 12 から続ける ]          │  ← 中断データがあるときのみ
│                                      │
│  ── ステージ ─────────────────────    │
│                                      │
│   [ 1✓] [ 2✓] [ 3✓] [ 4✓] [ 5✓]     │  ← ✓ = クリア済み
│   [ 6✓] [ 7 ] [ 8 ] [ 9 ] [10 ]     │  ← 数字 = 解放済み
│   [11 ] [12 ] [13 ] [14 ] [15 ]     │
│   [16░] [17░] [18░] ...             │  ← ░ = 未解放（非活性）
│        （縦スクロール）                 │
│                                      │
├──────────────────────────────────────┤
│ ┌──────────────────────────────────┐ │
│ │         バナー広告                │ │  ← 常時表示（モック時はプレースホルダ）
│ └──────────────────────────────────┘ │
└──────────────────────────────────────┘
```

| # | 要素 | 挙動 |
|---|------|------|
| 1 | スタミナ | `現在/3`。上限時は回復タイマー非表示 |
| 2 | リワードボタン | スタミナ &lt; 3 かつ 本日視聴 &lt; 5 のとき活性 |
| 3 | 続きから | `stageProgress` があるときのみ表示 |
| 4 | ステージボタン | タップでゲーム画面へ（スタミナ1消費。1〜5は無料） |
| 5 | バナー | 画面下部固定 |
| 6 | ステージ一覧 | **5列グリッド**、縦スクロール（確定） |


## 3. ゲーム画面

### 3.1 通常プレイ中

```
┌──────────────────────────────────────┐
│ [ ← ]    ステージ 7                    │  ← 戻る（中断・オートセーブ）
├──────────────────────────────────────┤
│                                      │
│              課題                      │
│           ┌──────┐                   │
│           │■青 ■赤│                   │  ← 画面中央寄せ（確定）
│           │■緑    │                   │
│           └──────┘                   │
│                                      │
│  NEXT [■黄]          残り 14 手       │
│                                      │
│  ┌──────────────────────────────┐    │
│  │ ·  ·  ·  ·  ·  │              │    │
│  │ ·  ■青 ■緑 ■赤 ■橙 │              │    │  ← 盤面（例: 5×5）
│  │ ·  ·  ■青 ■青 ■紫 │              │    │
│  │ ·  ·  ·  ·  ·  │              │    │
│  │ ·  ·  ·  ·  ·  │              │    │
│  └──────────────────────────────┘    │
│   ░ = お邪魔ブロック（グレー・模様）      │
│                                      │
├──────────────────────────────────────┤
│  行動を選ぶ（直前: B は不可）             │  ← 連続同一行動の制限を表示
│                                      │
│  [ A 混合 ]  [ B スライド ]  [ C 入替 ] │
│                                      │
└──────────────────────────────────────┘
 ※ バナー広告なし
```

| # | 要素 | 説明 |
|---|------|------|
| 1 | 戻る | メイン画面へ。プレイ状態を保存 |
| 2 | 課題 | クリアに必要な配色パターン（回転なし）。**画面中央寄せ**で表示（確定） |
| 3 | NEXT | 次回 B 実行時に出現する色 |
| 4 | 残り手 | `remainingTurns` |
| 5 | 盤面 | ステージサイズ可変。タップ操作の対象 |
| 6 | 行動ボタン | 直前と同じ行動は非活性 |

---

### 3.2 行動 A（混合）選択中

**2タップで即実行**（確定）。確認ボタンは出さない。

```
│  [ A 混合 ]  [ B スライド ]  [ C 入替 ] │
│       ↑ 選択中                         │
│                                      │
│  ┌──────────────────────────────┐    │
│  │ ·  ·  ·  ·  ·  │              │    │
│  │ ·  [■青][■黄] ■赤 ■橙 │  ← 2つ目タップで即混合
│  │ ·  ·  ·  ·  ·  │              │    │
│  └──────────────────────────────┘    │
```

- 1つ目タップ → ハイライト
- 2つ目タップ（隣接かつ混合可）→ **即実行**、ターン消費
- 2つ目が非隣接・混合不可 → トースト等で拒否、1つ目選択を解除
- 同じ行動を連続不可のため、直前が A なら A ボタン自体が非活性
- 選択中に A ボタン再タップ、または盤外タップで選択キャンセル（ターン消費なし）

---

### 3.3 行動 B（スライド）方向選択中

**方向パッド**で方向指定（確定）。方向ボタンタップで即スライド実行。

```
│  [ A 混合 ]  [ B スライド ]  [ C 入替 ] │
│                    ↑ 選択中             │
│                                      │
│            [ ▲ ]                     │
│        [ ◀ ][ · ][ ▶ ]               │  ← 方向パッド（タップで即実行）
│            [ ▼ ]                     │
│                                      │
│  （B 再タップでキャンセル）              │
```

- B ボタンタップ → 方向パッド表示
- 方向タップ → **即スライド実行** → 反対辺に NEXT 色が出現 → NEXT をランダム更新
- B 再タップでキャンセル（ターン消費なし）

---

### 3.4 行動 C（スワップ）選択中

**2タップで即実行**（確定）。確認ボタンは出さない。

```
│  [ A 混合 ]  [ B スライド ]  [ C 入替 ] │
│                              ↑ 選択中  │
│                                      │
│  ┌──────────────────────────────┐    │
│  │ ·  ·  ·  ·  ·  │              │    │
│  │ ·  [■緑][■赤] ■橙 · │  ← 2つ目タップで即入替
│  │ ·  ·  ·  ·  ·  │              │    │
│  └──────────────────────────────┘    │
```

- 1つ目タップ → ハイライト
- 2つ目タップ（隣接する操作可能ブロック）→ **即入替**、ターン消費
- 2つ目が非隣接・お邪魔ブロック → 拒否、1つ目選択を解除
- C 再タップ、または盤外タップで選択キャンセル（ターン消費なし）

### 3.5 クリア時（光多め演出 → オーバーレイ → メインへ）

専用画面は作らず、**光を中心とした派手めの演出**の後、オーバーレイで結果表示してメイン画面へ遷移（確定）。

#### 演出シーケンス（約 1.0〜1.5 秒）

```
(1) 一致マスが順に発光          (2) 課題パターンが光る
┌──────────────────────┐      ┌──────────────────────┐
│      [課題]          │      │    ✦ [課題] ✦       │  ← グロー拡大
│  ┌────────────┐      │      │  ┌────────────┐      │
│  │ · [✦][✦] · │      │  →   │  │ · [✦][✦] · │      │  ← 盤上の一致マスが光る
│  │ ·  ·  ·  · │      │      │  │░░░░░░░░░░░░│      │  ← 背景が明るくフラッシュ
│  └────────────┘      │      │  └────────────┘      │
└──────────────────────┘      └──────────────────────┘

(3) 「クリア！」が光の輪とともに表示
┌──────────────────────────────────────┐
│ ░░░░░░░░░ 白〜黄の放射グロー ░░░░░░░░ │  ← 画面全体がまだ明るい
│                                      │
│          ╭──────────────╮            │
│          │  ✦ クリア！ ✦ │            │  ← スプリングイン + グロー
│          │   [ OK ]      │            │
│          ╰──────────────╯            │
│         （盤面は背面に薄く表示）         │
└──────────────────────────────────────┘
        ↓ OK タップ（または約1秒後に自動）
     メイン画面（次ステージ解放反映）
```

| 段階 | 内容 | 技術（参考） |
|------|------|-------------|
| 1 | 一致マスが順番にパッと光る | Reanimated（scale + opacity） |
| 2 | 課題パターンが拡大しながら発光 | Reanimated |
| 3 | 背景が一瞬明るくなる（放射状グラデ） | 半透明 View のフェード |
| 4 | 「クリア！」＋光の輪で表示 | Reanimated + グロー用シャドウ |
| 5 | 軽い振動（任意） | expo-haptics |

- 演出中は操作ロック（タップ不可）。
- ゲームオーバーは光演出なし・簡素なオーバーレイのみ（§3.6）。

---

### 3.6 ゲームオーバー時（オーバーレイ → メインへ）

**オーバーレイで結果表示**してからメイン画面へ遷移（確定）。

```
┌──────────────────────────────────────┐
│          ┌────────────────┐          │
│          │ ゲームオーバー   │          │
│          │ 手数が尽きました  │          │  ← or「盤面がいっぱいです」
│          │  [ メインへ ]    │          │
│          └────────────────┘          │
└──────────────────────────────────────┘
```

- プレイ中データは削除
- メインから再挑戦時はスタミナ再消費（チュートリアル除く）

## 4. 画面遷移まとめ

| 操作 | 遷移 | 備考 |
|------|------|------|
| ステージタップ | メイン → ゲーム | スタミナ -1（1〜5除く） |
| 続きから | メイン → ゲーム | スタミナ消費なし |
| 戻る | ゲーム → メイン | オートセーブ |
| クリア | ゲーム → メイン | 次ステージ解放 |
| ゲームオーバー | ゲーム → メイン | セーブ削除 |
| リワード視聴完了 | （メイン上） | スタミナ +1 |

## 5. レイアウト方針（実装時の参考）

| 項目 | 方針 |
|------|------|
| 向き | 縦画面（ポートレート）固定想定 |
| セーフエリア | ノッチ・ホームインジケータを考慮 |
| 盤面 | 画面幅に合わせて正方形に近いグリッド。課題パターンの下に配置 |
| 課題パターン | 画面上部エリアの**中央寄せ** |
| ステージ一覧 | **5列**グリッド |
| 行動 A / C | **2タップで即実行**（確認ボタンなし） |
| 行動 B | **方向パッド**、方向タップで即実行 |
| クリア / GO | **オーバーレイ必須**（専用画面なし） |
| クリア演出 | **光多め・派手目**（一致マス発光→課題ハイライト→画面フラッシュ→「クリア！」） |
| タップ領域 | 行動ボタン・マスは最低 44pt 相当 |
| 色 | ブロックは色名に対応した塗り。お邪魔はグレー＋模様 |
| 非活性行動 | 直前と同じ行動ボタンはグレーアウト＋タップ不可 |

## 6. 初期リリース対象外（ワイヤーなし）

以下は要件上「未確定」または対象外。必要になったら追記する。

- 設定画面（BGM / SE トグルなど）
- チュートリアル専用オーバーレイ（ステージ1の説明ポップアップ）
- スコア・星評価の詳細 UI
- ストアページ用スクリーンショット構成

## 7. UI 確定事項

| 項目 | 決定内容 |
|------|----------|
| ステージ一覧 | 5列グリッド |
| 行動 A / C | 2タップで即実行（確認ボタンなし） |
| 行動 B | 方向パッド、方向タップで即実行 |
| クリア / ゲームオーバー | オーバーレイで表示後、メイン画面へ遷移 |
| 課題パターン | 画面中央寄せ。盤面はその下（相対位置は現ワイヤーのとおり） |
| クリア演出 | 光多め（グロー・フラッシュ中心）。紙吹雪は使わない |
